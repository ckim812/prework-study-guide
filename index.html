Vocabulary

  • HTML: hypertext markup language
  • CSS: cascading style sheets
  • Agile: adaptiveness and response to change
  • SCRUM: process framework to manage product development and other knowledge work
  • CLI: command line interface
  • SSH Key: access credential for the secure shell (SSH) network protocol
  • Web Development Methodologies: 1) Waterfall: each step is completed sequentially; better suited for smaller projects 2) Agile: project is broken down into smaller components which are build on an incremental basis
  • MVP: minimum viable product
  • SHA: "hash" - a long string of #'s or letters (developers often use the first 7 characters which give the right amount of information to allow for the right version to be retrieved and eliminates the need to copy the entire SHA)
  • Repository ("repo"): folder for your projects
  • Branch: workspace where you can make changes that don't affect the live site
  • Markdown (.md): the way content files are formatted in GitHub; a way to unite for web using plain text; easily converts to HTML
  • Commit Changes: saving a new version of a file you are working on (add/edit/delete); addes changes to your branch
  • Pull Request ("PR"): asking for changes on a branch to merge into another branch; a way to discuss and review your work with others
  • Merge: once a PR is approved, changes are "pulled in" or merged into the main branch; changes publish/deploy to the live site
  • Issue: a way to report new tasks or content fixes; track progress on progress board
